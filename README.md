# Парсер отзывов с Яндекс Карт (Laravel 11 + Vue 3)

Приложение собирает отзывы об организации по ссылке с Яндекс Карт, кэширует их в БД и показывает на фронте с фильтрацией и пагинацией.

## Как запустить

Всё работает в докере, поэтому поднимается одной командой.

1. Скопируйте конфиг бэкенда (настройки подключения к БД уже прописаны под докер):
```bash
cp backend/.env.example backend/.env
```

2. Соберите и поднимите контейнеры:
```bash
docker-compose up -d --build
```

3. Накатите миграции и сидер для тестового юзера:
```bash
docker compose exec backend php artisan migrate --seed
```

4. Откройте в браузере **http://localhost**.

Данные для входа:
* **Логин:** `test@example.com`
* **Пароль:** `password`

## Как устроен:

### 1. Парсинг через Puppeteer
Парсер на Node.js + Puppeteer.
* Скрипт запускает Chromium в headless-режиме прямо внутри контейнера бэкенда.
* Он находит скролл-контейнер отзывов на странице организации и имитирует прокрутку вниз до тех пор, пока не загрузит все отзывы (стоит лимит до 600 штук) или пока список не закончится.
* Для каждого отзыва генерируется хэш `md5(автор + текст + дата)`.

### 2. Очереди задач
Headless-браузер скроллит страницу около 20-40 секунд. Если делать это прямо в HTTP-запросе, клиент поймает таймаут 504.
* Я вынес парсинг в фоновую джобу `ScrapeYandexReviewsJob`.
* При сохранении ссылки бэкенд сразу создает запись со статусом `pending` и кидает джобу в очередь, отдавая ответ фронту за миллисекунды.
* Фронтенд раз в 3 секунды опрашивает статус (polling). Как только статус сменился на `completed`, подгружаются данные.

### 3. Чистая архитектура бэкенда
* Валидация вынесена в Form Requests, чтобы следовать принципу тонких контроллеров.
* Внедрен паттерн Репозиторий. Контроллер общается с БД исключительно через интерфейсы репозиториев. Это избавляет от жесткой привязки к Eloquent и делает код тестируемым.
* Для записи отзывов в БД используется транзакция. Если при вставке пачки отзывов произойдет сбой, база вернется в исходное состояние без мусора.

## Что можно улучшить, если бы было больше времени

1. **Прокси:** Для продакшена обязательно нужно прикрутить пул мобильных/резидентских прокси для Puppeteer, иначе Яндекс быстро забанит IP сервера при частых запросах.
2. **Антикапча:** Интегрировать сервисы вроде rucaptcha/2captcha на случай, если Яндекс выдаст капчу при переходе на страницу.
3. **WebSockets:** Заменить опрос статуса (polling) на веб-сокеты (через Laravel Reverb или Pusher), чтобы статус обновлялся мгновенно без лишних запросов.
4. **Пагинация на бэкенде:** Сейчас пагинация отзывов (по 50 штук) работает на бэкенде через БД, что очень быстро. Но сам парсер тянет всю пачку до 600 штук разом. При масштабировании на тысячи отзывов нужно будет парсить порциями.
